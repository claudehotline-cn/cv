## Architecture

The Transformer architecture, introduced in the 2017 paper "Attention Is All You Need" by Google researchers led by Ashish Vaswani, represents a foundational shift in neural network design for sequence modeling. It processes text through a multi-head attention mechanism, enabling efficient handling of long-range dependencies while eliminating the sequential limitations of traditional recurrent architectures. Key steps include:

- **Input Processing**: Text is converted into tokens, each mapped to a vector via word embedding (e.g., a 512-dimensional space for standard implementations), where the embedding table provides numerical representations for discrete tokens.
- **Contextualization**: Tokens are contextualized within a context window (e.g., 512 tokens) using parallel multi-head attention, amplifying signals for key tokens while diminishing less important ones. This allows the model to maintain signal integrity across sequences up to 10,000 tokens, unlike RNNs that degrade for sequences exceeding 50 tokens due to information loss.
- **Parallel Processing**: Unlike RNNs, Transformers process all tokens in parallel, eliminating sequential dependencies. This parallelizability reduces training time significantly; studies show it requires 6–7× less training time than LSTM-based models for equivalent tasks, as noted in comparative analyses.

A critical advantage is the absence of recurrent units (e.g., no LSTM or GRU layers), which avoids the vanishing gradient problem inherent in RNNs. For instance, in a 2016 comparison, RNN-based models struggled with sequences exceeding 50 tokens, whereas Transformers maintain robust performance across extended context windows. The architecture comprises two main components: the encoder and the decoder, both leveraging self-attention and positional encoding.

The encoder processes the input sequence (e.g., source language text) into contextualized representations using self-attention. It consists of 6 layers, each with 4 attention heads, and incorporates positional encoding (e.g., sinusoidal functions) to inject token order information, as the model lacks inherent sequence awareness. Historically, LSTM encoders suffered from information loss for long inputs; a 2017 study showed performance degradation by 15–20% when sequences exceeded 100 tokens due to fixed-size vector constraints. The encoder generates a 512-dimensional vector per token within a 512-token context window, ensuring bidirectional dependency capture.

The decoder generates the output sequence (e.g., target language text) using masked multi-head attention to prevent future token leakage and cross-attention to connect to the encoder. It also uses 6 layers, processing tokens in parallel. Early RNN decoders processed tokens sequentially, but the Transformer's parallel processing achieves higher BLEU scores; for a 1000-token sequence, it achieves 22% higher BLEU scores than LSTM decoders due to efficient context utilization. Notably, before the 2017 paper, co-author Jakob Uszkoreit applied a "decoder-only" architecture to generate fictitious Wikipedia articles, demonstrating its utility for text generation.

The attention mechanism allows the model to weigh token importance via dot products (e.g., query-key similarity), computing a weighted sum of token vectors. Early seq2seq models lacked attention, leading to poor performance on long sequences (e.g., reversing input sentences improved translation quality by 12%, indicating information loss). The Transformer fully replaces recurrent units with attention, enabling superior performance across diverse tasks. This architecture revolutionized sequence modeling by leveraging parallel processing and attention, setting new standards in machine translation and text generation, with empirical evidence supporting its efficiency and scalability.
