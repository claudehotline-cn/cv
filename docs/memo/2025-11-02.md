2025-11-02  Docker 构建开启 CUDA 内核/NVDEC/NVENC 与 ORT GPU

- 目标：在 VA 的 docker 构建中开启 CUDA 核函数（decode/NMS/overlay）、NVDEC/NVENC，以及 ONNX Runtime GPU（启用 IoBinding）。
- 变更：
  - docker/va/Dockerfile.gpu：
    - build 阶段：安装 curl；下载并解压 onnxruntime-linux-x64-gpu（默认 1.17.1）到 /opt/onnxruntime；
      cmake 增加 -DWITH_CUDA_KERNELS=ON -DWITH_NVDEC=ON -DWITH_NVENC=ON -DONNXRUNTIME_ROOT=/opt/onnxruntime。
    - runtime 阶段：复制 /opt/onnxruntime；设置 LD_LIBRARY_PATH=/opt/onnxruntime/lib；补齐 OpenCV videoio/highgui/imgcodecs 等运行库。
  - 依赖：此前已补 cp/vsm/va 运行时缺失库（libgrpc++1、OpenCV 组件）。
- 现状：
  - 构建镜像将编译并链接 CUDA 核与 NVDEC/NVENC；
  - ORT GPU 头+库可用，USE_ONNXRUNTIME 宏启用后，ORT CUDA EP 与 IoBinding 可用（CMake 通过 ONNXRUNTIME_ROOT 检测）。
- 后续：
  - 如需固定 ORT 版本，可在 compose 中通过 build-arg ORT_VERSION 覆盖；
  - 若需 CPU 镜像也开启 ORT（非 GPU），可在 Dockerfile.cpu 中对接 onnxruntime CPU 包；
  - 运行时需具备 NVIDIA 驱动与容器工具包，FFmpeg NV 编解码可用。


- Param: 将 Dockerfile.gpu 的 CUDA/Ubuntu 版本参数化（CUDA_VER/UBUNTU_VER），并在 compose.gpu.override.yml 中传入，便于与你宿主机 CUDA 版本保持一致。

- CUDA: 根据 nvcc --version (12.9.86)，已在 compose.GPU override 中将 CUDA_VER 设为 12.9.0（若远端无该标签，可回退到 12.8.x/12.6.x）。
