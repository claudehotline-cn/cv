2025-11-01  当前进展与修改记录

- 任务：修复容器内 VA 构建失败（ixwebsocket 头缺失、WHEP 源重复定义）。
- 变更：
  - 为 RTC 相关源码添加条件编译与存根：
    - video-analyzer/src/media/transport_webrtc_datachannel.cpp
      - 在 VA_HAVE_RTC==1 时保留原实现；在 VA_HAVE_RTC==0 时提供最小存根（connect/send/断开/统计），避免依赖 ixwebsocket/libdatachannel。
    - video-analyzer/src/media/whep_session.cpp
      - 用 `#if defined(VA_HAVE_RTC) && VA_HAVE_RTC` 包裹实现，避免与 whep_session.hpp 的 `!VA_HAVE_RTC` 内联存根发生重定义。
  - 目的：消除 Docker 构建日志中的两类错误：
    - fatal error: ixwebsocket/IXWebSocketServer.h: No such file or directory
    - redefinition of WhepSessionManager::* 与缺失 rtc::* 符号
- 预期影响：
  - 无 RTC 头文件/库的环境（容器）可成功编译 VA；WHEP 仍由 CP 代理路径负责，RTC 数据通道默认禁用。
  - Windows/vcpkg 完整环境仍可启用 RTC（后续可在 CMake 中按有无头文件自动开启 VA_HAVE_RTC）。
- 后续建议：
  - 如需在容器启用真实 WHEP 媒体：为 VA 镜像安装 libdatachannel 与 ixwebsocket 头库，并在 CMake 中探测开启 VA_HAVE_RTC=1，同时仅在可用时编译相关源文件。
  - 统一 CMake 对 VA_HAVE_RTC 的探测（基于 `check_cxx_source_compiles` 或 `check_include_file_cxx`），去除无条件 `-DVA_HAVE_RTC=0` 的剩余路径。
