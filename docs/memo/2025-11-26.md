  - 在 `controlplane-spring/pom.xml` 中引入 `spring-boot-starter-aop`、`resilience4j-spring-boot3` 与 `micrometer-registry-prometheus`，并配置 `protobuf-maven-plugin` 基于 `src/main/proto` 生成 `va.v1` 与 `vsm.v1` 的 gRPC Java stub，修复了之前 Docker 构建阶段缺少 Resilience4j 版本和未生成 stub 导致的编译错误。

  - 调整 `SubscriptionServiceImpl` 与 `SourceServiceImpl`：在订阅/取消订阅、源 attach/detach/enable/disable 时通过上述 gRPC 客户端调用 VA/VSM，同步维护内存态 `SubscriptionRecord` / `SourceItem` 映射，使 `/api/subscriptions` 与 `/api/sources*` 的调用链真正贯通到下游服务，同时保留对本地状态的 ETag、列表等行为支持。
  - 新增 `GlobalExceptionHandler`（`@RestControllerAdvice`）：统一捕获 `StatusRuntimeException`，根据 gRPC `Status.Code` 映射为 HTTP 400/404/409/503/500，并返回 `CpResponse` 结构体中的 `code/msg` 字段；其余未捕获异常统一映射为 `INTERNAL`，避免 gRPC 异常栈直接泄露到客户端。
