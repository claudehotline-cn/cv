- 结论：
  - 更新 `controlplane/test/scripts/check_cp_sources_attach_detach.py`，新增 `CP_TEST_RTSP_URI` 环境变量支持：默认仍使用原来的 `rtsp://127.0.0.1:8554/camera_01`，但如果在环境中设置了 `CP_TEST_RTSP_URI`，脚本会将其用作 `source_uri`，并通过 `urllib.parse.quote` 编码后拼入 `/api/sources:attach` 的查询字符串。
  - 这样在 Docker 场景下可以将 `CP_TEST_RTSP_URI` 设置为 `rtsp://host.docker.internal:8554/camera_01` 或 `rtsp://192.168.50.78:8554/camera_01` 等容器可达的地址，避免 VSM 在容器内误连 `127.0.0.1`（指向容器自身）而无法拉到宿主机 RTSP 流，从而更准确地验证 cp-spring ↔ VSM ↔ RTSP 的真实链路。
  - 在非 Docker 或本地单进程开发环境中，若不设置 `CP_TEST_RTSP_URI`，脚本行为保持与之前完全一致，对现有使用方式不产生破坏；后续若需要，也可以在其它 CP/VSM 相关测试脚本中复用这一模式，使测试地址在不同部署环境下更易于配置与切换。
