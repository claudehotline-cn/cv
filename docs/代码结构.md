# 代码结构总览

本文档描述当前仓库的目录划分与各模块职责，便于在 GPU 全链路改造过程中保持实现与文档一致。

## 根目录布局

```
cv/
├── CMakeLists.txt              # 顶层构建脚本（暂仅聚焦 video-analyzer）
├── docs/                       # 设计与改造文档
├── logs/                       # 运行时日志默认输出目录
├── third_party/                # 预编译依赖或本地源码镜像
├── tools/                      # 研发辅助脚本（不纳入版本控制）
├── video-analyzer/             # 后端推理与流媒体服务
├── video-source-manager/       # （暂保留）视频源管理项目
└── web-frontend/               # Web 前端（Vue + Vite）
```

除 README.md 外，所有文档均位于 `docs/`，其中 `全链路GPU改造/` 子目录存放 GPU 改造阶段性方案。

## video-analyzer 模块

```
video-analyzer/
├── CMakeLists.txt
├── config/                 # YAML 配置：engine、models、profiles 等
├── include/                # 对外暴露的头文件（与 src/* 同步）
├── src/
│   ├── analyzer/           # 预处理、推理会话、后处理、渲染
│   ├── app/                # 应用启动与运行态管理
│   ├── core/               # Pipeline、EngineManager、BufferPool 等核心抽象
│   ├── media/              # 解码、编码、传输实现（DataChannel/WHIP 占位）
│   ├── server/             # REST 服务与路由
│   └── composition_root.cpp# 组装所有工厂与依赖
├── build/                  # CMake 构建产物（Release/Debug）
├── test/                   # 后端测试脚本与检查工具
└── model/、data/           # 示例模型或演示资源
```

- **core/utils.(hpp|cpp)**：定义 `Frame`、`FrameSurface`、`TensorView` 等通用数据结构。
- **core/pipeline.cpp**：拉流 → 分析 → 编码 → 传输的主循环，Stage 7 中逐步切换到 `FrameSurface`。
- **analyzer/**：`LetterboxPreprocessor*`、`OrtModelSession`、`Postprocessor` 与 `Renderer` 等组件组成推理链路。
- **media/**：`SwitchableRtspSource` 基于 OpenCV/FFmpeg 拉流；`FfmpegH264Encoder` 负责编码；`transport_webrtc_datachannel` 提供 WebRTC 数据通道。`transport_whip.*` 目前为 WHIP/WHEP 占位实现。
- **server/rest.cpp**：提供 `/subscribe`、`/switch`、`/engine` 等 HTTP 接口，调用 `TrackManager`/`EngineManager` 协调资源。

## 其他模块

- **video-source-manager**：历史项目，暂未参与 GPU 改造，保留以便后续迁移。
- **web-frontend**：Vue3 + TypeScript 前端，对接 REST/WebRTC，后续需展示 GPU 运行态信息。

后续所有代码改动应与本文档保持一致，如目录或职责发生调整，请同步更新此文件与相关设计文档。
