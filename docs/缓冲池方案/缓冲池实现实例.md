太好了。我把**两个最关键的补丁**一次性给全：

1. `OrtModelSession::run()` 的**池化 + IOBinding** 最小实现；
2. **自定义 OrtAllocator**（CPU / CudaPinned / Cuda 三种域）注册到 Env，并启用 `session.use_env_allocators=1`。
    这样你们现有的 `HostBufferPool/GpuBufferPool` 就真正“接入” ONNX Runtime 的 I/O 了：**已知形状**零拷贝复用池、**动态形状**先走 BindOutputToDevice，等 L2 完成后 ORT 也会直接向你的池要内存。参考点位都附在文末（IOBinding、Device tensors、注册自定义 allocator、use_env_allocators 等）。[ONNX Runtime+4ONNX Runtime+4ONNX Runtime+4](https://onnxruntime.ai/docs/performance/tune-performance/iobinding.html)

# 补丁 A：`OrtModelSession`（或你们的 `OrtSession`）最小实现

> 下面假设你的类名是 `OrtModelSession`，如果你仓库里是 `OrtSession`，把名字替换即可。
>  关键点：
>
> - 输入/输出都用 **IOBinding**（C++ API 支持两种绑定：绑定具体 `Ort::Value` 或绑定 **MemoryInfo** 让会话分配）。[ONNX Runtime](https://onnxruntime.ai/docs/api/c/struct_ort_1_1_io_binding.html)
> - **GPU 输入/输出**：用 `Ort::MemoryInfo("Cuda", OrtArenaAllocator, device_id, OrtMemTypeDefault)` + `CreateTensor` 包装**你池里分配的指针**（不拷贝）。[ONNX Runtime](https://onnxruntime.ai/docs/performance/device-tensor.html)
> - **动态输出**：先用 `io.BindOutput(name, mem_info)`（只绑定域），会话分配；L2 完成后会话分配也会走你的池（见补丁 B）。[ONNX Runtime](https://onnxruntime.ai/docs/performance/tune-performance/iobinding.html)

**`src/analyzer/ort_session.hpp`（增量）**

```
#pragma once
#include <onnxruntime_cxx_api.h>
#include "core/utils.hpp"
#include "core/buffer_pool.hpp"
#include <memory>
#include <string>
#include <vector>

class OrtModelSession {
public:
  struct Options {
    bool   use_iobinding = true;
    size_t tensor_host_pool_bytes   = 0;  // 开启即建Host池（可选：分块策略由池实现决定）
    size_t tensor_device_pool_bytes = 0;  // 开启即建GPU池
    bool   use_env_allocators = true;     // L2: 让会话用Env注册的allocator
    int    cuda_device = 0;
    bool   outputs_may_vary = false;      // 动态输出形状
  };

  OrtModelSession(const std::string& onnx, const Options& opt);

  // x: Device/Host 皆可（由 MemoryHandle::location 决定）
  bool run(const core::TensorView& x,
           std::vector<core::TensorView>& ys,   // 输出张量视图（优先Device）
           std::vector<std::string> output_names = {}); // 为空则用模型默认输出顺序

private:
  Ort::Env env_{ORT_LOGGING_LEVEL_WARNING, "va-ort"};
  Ort::Session sess_{nullptr};
  Options opt_;
  // 本地输出缓冲池（L1手动接入；L2后可弱化为仅作回退）
  std::unique_ptr<core::HostBufferPool> host_pool_;
  std::unique_ptr<core::GpuBufferPool>  gpu_pool_;

  // 小工具
  static Ort::MemoryInfo mi_cuda(int dev) {
    return Ort::MemoryInfo("Cuda", OrtArenaAllocator, dev, OrtMemTypeDefault);
  }
  static Ort::MemoryInfo mi_cpu() {
    return Ort::MemoryInfo::CreateCpu(OrtArenaAllocator, OrtMemTypeDefault);
  }
};
```

**`src/analyzer/ort_session.cpp`（完整最小版）**

```
#include "ort_session.hpp"
#include <stdexcept>
using namespace core;

OrtModelSession::OrtModelSession(const std::string& onnx, const Options& opt) : opt_(opt) {
  Ort::SessionOptions so;
  // 供 L2 使用：让 Session 使用 Env 注册的 allocator
  if (opt_.use_env_allocators) {
    so.AddConfigEntry("session.use_env_allocators", "1"); // 官方文档配置键 :contentReference[oaicite:4]{index=4}
  }
  // 启用 CUDA EP（或 TRT EP），此处以 CUDA 为例
  {
    OrtCUDAProviderOptions cuda_opts{};
    cuda_opts.device_id = opt_.cuda_device;
    OrtSessionOptionsAppendExecutionProvider_CUDA(so, cuda_opts);
  }
  sess_ = Ort::Session(env_, onnx.c_str(), so);

  // L1：根据配置初始化本地池
  if (opt_.tensor_device_pool_bytes > 0) {
    // 例：按固定块大小预分配若干块 Device NV12/tensor 缓冲；这里仅存根
    gpu_pool_ = std::make_unique<GpuBufferPool>(/*capacity*/8, /*w*/0, /*h*/0, PixelFormat::RGBA32F);
  }
  if (opt_.tensor_host_pool_bytes > 0) {
    host_pool_ = std::make_unique<HostBufferPool>(/*capacity*/8, /*bytes*/opt_.tensor_host_pool_bytes/8);
  }
}

bool OrtModelSession::run(const TensorView& x, std::vector<TensorView>& ys, std::vector<std::string> output_names) {
  // 1) 输入：从 handle 封装 Ort::Value（不拷贝）
  Ort::IoBinding io(sess_);
  std::array<int64_t, 4> in_shape{}; // 假设 NCHW，真实从 x.shape 取
  in_shape.assign(x.shape.begin(), x.shape.end());

  Ort::Value in_val{nullptr};
  if (x.handle.location == MemoryLocation::Device) {
    auto mi = mi_cuda(opt_.cuda_device);
    in_val = Ort::Value::CreateTensor(mi,
              reinterpret_cast<void*>(x.handle.device_ptr),
              x.handle.bytes, in_shape.data(), (size_t)in_shape.size(),
              ONNX_TENSOR_ELEMENT_DATA_TYPE_FLOAT); // 或据 dtype 映射
  } else {
    auto mi = mi_cpu();
    in_val = Ort::Value::CreateTensor(mi,
              x.handle.host_ptr, x.handle.bytes,
              in_shape.data(), (size_t)in_shape.size(),
              ONNX_TENSOR_ELEMENT_DATA_TYPE_FLOAT);
  }
  // 绑定输入
  // 假设唯一输入名为 "images"，实际应从 sess_ 的输入元信息读取
  io.BindInput("images", in_val);  // IOBinding C++ API（绑定具体 Value） :contentReference[oaicite:5]{index=5}

  // 2) 输出：两条路——已知形状(预分配) vs 动态形状(绑定域)
  if (output_names.empty()) {
    // 若未显式给出，取模型输出名
    Ort::AllocatorWithDefaultOptions def_alloc;
    std::vector<Ort::Value> dummy;
    // 通过 sess_ 元信息或提前缓存来得到输出名；此处为简化略去细节
  }

  if (!opt_.outputs_may_vary && gpu_pool_) {
    // 2.a 已知形状：从 GPU 池拿块 -> CreateTensor -> BindOutput(value)
    for (const auto& name : output_names) {
      MemoryHandle h{};
      if (!gpu_pool_->acquire(h)) throw std::runtime_error("gpu pool depleted");
      // TODO: 设定 h.bytes / h.device_ptr 为合适大小
      auto v = Ort::Value::CreateTensor(mi_cuda(opt_.cuda_device),
                reinterpret_cast<void*>(h.handle.device_ptr),
                h.bytes, /*out_shape*/nullptr, /*ndim*/0, ONNX_TENSOR_ELEMENT_DATA_TYPE_FLOAT);
      io.BindOutput(name.c_str(), v); // 绑定用户缓冲（零拷贝） :contentReference[oaicite:6]{index=6}
      // 运行后把 h 封进 ys[i].handle，生命周期由调用者管理，归还时 release()
      TensorView y{}; y.handle = h; ys.push_back(y);
    }
  } else {
    // 2.b 动态形状或无 GPU 池：只绑定 MemoryInfo（让会话分配）
    for (const auto& name : output_names) {
      auto info = (x.handle.location == MemoryLocation::Device)
                  ? mi_cuda(opt_.cuda_device) : mi_cpu();
      io.BindOutput(name.c_str(), info); // BindOutputToDevice/Host（动态输出推荐） :contentReference[oaicite:7]{index=7}
    }
  }

  // 3) 运行
  Ort::RunOptions run_opt;
  sess_.Run(run_opt, io); // IOBinding 模式运行（输入/输出预分配或按域分配） :contentReference[oaicite:8]{index=8}

  if (opt_.outputs_may_vary || !gpu_pool_) {
    // 动态形状：取回 ORT 分配的输出，再“接回”你们的池（L2 完成后可省掉复制）
    auto out_vals = io.GetOutputValues(); // 取所有输出值（类型/shape都在里头） :contentReference[oaicite:9]{index=9}
    ys.clear(); ys.reserve(out_vals.size());
    for (auto& ov : out_vals) {
      auto type = ov.GetTensorTypeAndShapeInfo();
      auto elem = type.GetElementType();
      auto shape = type.GetShape();
      size_t bytes = type.GetElementCount() * sizeof(float); // 依据 elem 决定

      if (x.handle.location == MemoryLocation::Device && gpu_pool_) {
        MemoryHandle h{};
        if (!gpu_pool_->acquire(h)) throw std::runtime_error("gpu pool depleted");
        // TODO: 把 ov 中的数据拷/搬到 h.device_ptr（若同为 Device，可用 cudaMemcpyAsync 或者尝试避免拷贝）
        TensorView y{}; y.shape.assign(shape.begin(), shape.end()); y.handle = h;
        ys.push_back(std::move(y));
        // NOTE: L2 完成后：直接让 ORT 用你的 allocator 分配到池，就不需要这步搬运了
      } else {
        MemoryHandle h{};
        if (host_pool_ && host_pool_->acquire(h)) {
          // 拷到 pinned host（D2H 异步可选）
          // memcpy(h.host_ptr, ov.GetTensorData<float>(), bytes);
          TensorView y{}; y.shape.assign(shape.begin(), shape.end()); y.handle = h;
          ys.push_back(std::move(y));
        } else {
          // 退化：直接把 ORT 的 host 缓冲封成一个只读 view（生命周期注意）
          // 实务中建议总是走池，便于统一释放
        }
      }
    }
  }

  return true;
}
```

>
> - IOBinding 的两种绑定方式（`BindOutput(name, Value)` vs `BindOutput(name, MemoryInfo*)`）是官方推荐的做法，动态输出建议优先**绑定域**，避免尺寸不匹配报错；官方也在 issue 中建议如此。[ONNX Runtime+2ONNX Runtime+2](https://onnxruntime.ai/docs/performance/tune-performance/iobinding.html)
> - **设备张量外部包装**用 `Ort::Value::CreateTensor(memory_info_cuda, device_ptr, ...)`，不发生拷贝，文档有明确示例。[ONNX Runtime](https://onnxruntime.ai/docs/performance/device-tensor.html)

# 补丁 B：把你的池变成 ORT 的 allocator（L2）

目标：在**动态输出**场景，也让 ORT 直接从你的池里分配；做法是注册**自定义 OrtAllocator**到 **Env**，并开启 `session.use_env_allocators=1`。[ONNX Runtime+1](https://onnxruntime.ai/docs/api/c/struct_ort_api.html)

**`src/analyzer/ort_pool_allocator.hpp`（新）**

```
#pragma once
#include <onnxruntime_c_api.h>
#include "core/buffer_pool.hpp"

// 为不同内存域各准备一个 allocator 实例：CPU / CudaPinned / Cuda
struct PoolAllocator {
  OrtAllocator base;            // 函数指针表（C风格）
  core::HostBufferPool* host_pool = nullptr;
  core::GpuBufferPool*  gpu_pool  = nullptr;
  OrtMemoryInfo* mi = nullptr;  // "Cpu"/"CudaPinned"/"Cuda"
};

// 工厂：创建+注册 allocator 到 Env
// 注意：注册后，所有使用同一 OrtEnv 的 Session 都可共享（官方语义）
OrtStatus* CreateAndRegisterCpuPinnedAllocator(OrtEnv* env, PoolAllocator** out,
                                              core::HostBufferPool* host_pool);
OrtStatus* CreateAndRegisterCudaAllocator(OrtEnv* env, PoolAllocator** out,
                                          core::GpuBufferPool* gpu_pool, int device);
```

**`src/analyzer/ort_pool_allocator.cpp`（新：最小 C 实现样板）**

```
#include "ort_pool_allocator.hpp"
#include <cstring>
#include <new>

static void* ORT_API_CALL PoolAlloc(void* this_ptr, size_t size) {
  auto* self = reinterpret_cast<PoolAllocator*>(this_ptr);
  core::MemoryHandle h{};
  if (self->mi && std::string(OrtGetApiBase()->GetApi(ORT_API_VERSION)->GetMemoryInfoName(self->mi)) == "Cuda") {
    if (!self->gpu_pool || !self->gpu_pool->acquire(h)) return nullptr;
    // 这里简化：假设池块 >= size；真实实现应检查/切换更大块或合并策略
    return reinterpret_cast<void*>(h.device_ptr);
  } else {
    if (!self->host_pool || !self->host_pool->acquire(h)) return nullptr;
    return h.host_ptr;
  }
}

static void ORT_API_CALL PoolFree(void* this_ptr, void* p) {
  auto* self = reinterpret_cast<PoolAllocator*>(this_ptr);
  // 归还：需要你们在池里支持通过指针定位块，这里仅示意
  core::MemoryHandle h{};
  if (self->mi && std::string(OrtGetApiBase()->GetApi(ORT_API_VERSION)->GetMemoryInfoName(self->mi)) == "Cuda") {
    h.device_ptr = reinterpret_cast<CUdeviceptr>(p);
    if (self->gpu_pool) self->gpu_pool->release(std::move(h));
  } else {
    h.host_ptr = p;
    if (self->host_pool) self->host_pool->release(std::move(h));
  }
}

static const OrtMemoryInfo* ORT_API_CALL PoolInfo(const void* this_ptr) {
  auto* self = reinterpret_cast<const PoolAllocator*>(this_ptr);
  return self->mi;
}

static void ORT_API_CALL PoolDestroy(void* this_ptr) {
  auto* self = reinterpret_cast<PoolAllocator*>(this_ptr);
  if (self->mi) OrtGetApiBase()->GetApi(ORT_API_VERSION)->ReleaseMemoryInfo(self->mi);
  delete self;
}

static OrtStatus* fill_allocator(PoolAllocator* pa, const char* name, int device_id, OrtMemType mem_type) {
  auto* api = OrtGetApiBase()->GetApi(ORT_API_VERSION);
  OrtStatus* st = nullptr;
  st = api->CreateMemoryInfo(name, OrtArenaAllocator, device_id, mem_type, &pa->mi);
  if (st) return st;
  pa->base.version = ORT_API_VERSION;
  pa->base.Alloc   = PoolAlloc;
  pa->base.Free    = PoolFree;
  pa->base.Info    = PoolInfo;
  pa->base.GetMemType = [](const void*) { return OrtMemTypeDefault; };
  pa->base.reserve = nullptr;
  return nullptr;
}

OrtStatus* CreateAndRegisterCpuPinnedAllocator(OrtEnv* env, PoolAllocator** out, core::HostBufferPool* host_pool) {
  auto* pa = new (std::nothrow) PoolAllocator{};
  if (!pa) return OrtGetApiBase()->GetApi(ORT_API_VERSION)->CreateStatus(ORT_RUNTIME_EXCEPTION, "OOM");
  pa->host_pool = host_pool;
  auto* api = OrtGetApiBase()->GetApi(ORT_API_VERSION);
  if (auto* st = fill_allocator(pa, "CudaPinned", /*device*/0, OrtMemTypeDefault)) return st;  // CudaPinned 域（Pinned Host）:contentReference[oaicite:13]{index=13}
  auto* st2 = api->RegisterAllocator(env, &pa->base); // 注册到 Env（多 Session 共享）:contentReference[oaicite:14]{index=14}
  if (st2) { PoolDestroy(pa); return st2; }
  *out = pa; return nullptr;
}

OrtStatus* CreateAndRegisterCudaAllocator(OrtEnv* env, PoolAllocator** out, core::GpuBufferPool* gpu_pool, int device) {
  auto* pa = new (std::nothrow) PoolAllocator{};
  if (!pa) return OrtGetApiBase()->GetApi(ORT_API_VERSION)->CreateStatus(ORT_RUNTIME_EXCEPTION, "OOM");
  pa->gpu_pool = gpu_pool;
  auto* api = OrtGetApiBase()->GetApi(ORT_API_VERSION);
  if (auto* st = fill_allocator(pa, "Cuda", device, OrtMemTypeDefault)) return st; // Cuda 域（Device）:contentReference[oaicite:15]{index=15}
  auto* st2 = api->RegisterAllocator(env, &pa->base); // 注册
  if (st2) { PoolDestroy(pa); return st2; }
  *out = pa; return nullptr;
}
```

## 集成顺序（与你计划一致）

1. **配置透传**：从 `AppConfigPayload → FilterConfig → OrtModelSession::Options` 开启 `tensor_host_pool_bytes/tensor_device_pool_bytes`。
2. **L1 完成**：
   - `run()` 输入/输出都走 **IOBinding**；
   - 已知形状输出：**从池预分配 → BindOutput(value)**；
   - 动态输出：**BindOutput(mem_info)** → `GetOutputValues()` → 先“接回”池（过渡期拷/搬一次）。[ONNX Runtime](https://onnxruntime.ai/docs/performance/tune-performance/iobinding.html)
3. **L2 完成**：
   - 注册 **PoolAllocator(CPU/CudaPinned/Cuda)** 到 **Env**；
   - 每个会话设置 `session.use_env_allocators=1`；
   - 动态输出路径无需再拷贝，ORT 会直接从你的池分配（含 IOBinding 的域分配）。[ONNX Runtime+1](https://onnxruntime.ai/docs/api/c/struct_ort_api.html)

## 验收点

- **无临时 std::vector/裸指针**：推理 I/O 全部来自 `HostBufferPool/GpuBufferPool`；
- **动态输出**不再尺寸不符（用 BindOutputToDevice/Host 规避）[GitHub](https://github.com/microsoft/onnxruntime/issues/8144?utm_source=chatgpt.com)；
- **零拷贝/少拷贝**：设备输入/输出均为 `CreateTensor(memory_info_cuda, device_ptr, ...)` 包装；
- **Pinned Host**：D2H 路径走 `"CudaPinned"` 域（可观察异步）。[ONNX Runtime](https://onnxruntime.ai/docs/performance/device-tensor.html)
