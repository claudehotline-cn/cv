### 与上一版的差异点（★）

- `analyzer/ort_model_session.{hpp,cpp}`：取代/合并原 `ort_session.*`，实现了
  - 输入/输出 **IOBinding**；
  - **已知形状**输出从 `GpuBufferPool/HostBufferPool` 预分配后 `BindOutput(value)`；
  - **动态形状**输出走 `BindOutput(MemoryInfo)`，随后（L2 完成后）由 ORT 直接用你们的池分配。
- `analyzer/ort_pool_allocator.{hpp,cpp}`：提供 **CPU / CudaPinned / Cuda** 三类 **OrtAllocator**，注册到 **Env**，并在会话中设置 `session.use_env_allocators=1`。
- `composition_root.cpp`：在创建会话前**注册 allocator**（Env 级别），并在 `SessionOptions` 开启 `use_env_allocators`；工厂里改为构造 `OrtModelSession`。

> 说明：如果你仓库里仍保留了历史文件 `analyzer/ort_session.*`，可以删除/重命名为上面的 `ort_model_session.*`，或在 `engine_manager` 中把 `OrtModelSession` 作为 `IModelSession` 的实现返回，保持对上层透明。



下面给你**可直接落库**的补丁：把**自定义内存池接入 ORT**（注册 allocator + 打开 `use_env_allocators`）、在 `composition_root.cpp` 里完成一次性注册，并让 `EngineManager` 用新的 `OrtModelSession` 构建会话（CUDA EP + IOBinding）。相关要点引用了 ONNX Runtime 官方文档：**IOBinding 两种绑定方式**、**设备张量/`CudaPinned` 域**、**注册自定义 allocator 到 Env 并在会话启用**、**CUDA EP**。[ONNX Runtime+4ONNX Runtime+4ONNX Runtime+4](https://onnxruntime.ai/docs/performance/tune-performance/iobinding.html?utm_source=chatgpt.com)

# 2) `composition_root.cpp`：注册自定义 allocator + 构建工厂

> 在**进程启动时**注册你们的 **Host（Pinned）/GPU** 池为 ORT 分配器（Env 级别），然后所有 Session 设置 `use_env_allocators=1` 即可使用。

```
#include "core/factories.hpp"
#include "core/engine_manager.hpp"
#include "core/buffer_pool.hpp"

#include "analyzer/preproc_letterbox_cpu.hpp"
#include "analyzer/preproc_letterbox_cuda.hpp"
#include "analyzer/ort_model_session.hpp"
#include "analyzer/ort_pool_allocator.hpp"   // ★ 自定义 ORT 分配器
#include "analyzer/postproc_yolov12_det.hpp"
#include "analyzer/postproc_transformer_det.hpp"
#include "analyzer/render_box.hpp"

#include "media/source_switchable_rtsp_cpu.hpp"
#include "media/source_switchable_rtsp_cuda.hpp"
#include "media/encoder_h264_ffmpeg.hpp"
#include "media/encoder_h264_nvenc.hpp"
#include "media/transport_whip.hpp"

// 全局/共享 ONNX Runtime Env（确保全局唯一）
static Ort::Env g_env{ORT_LOGGING_LEVEL_WARNING, "va-ort"};

// 供 Env allocator 使用的共享池（示例：可按你的配置尺寸初始化）
static std::unique_ptr<core::HostBufferPool> g_host_pool;
static std::unique_ptr<core::GpuBufferPool>  g_gpu_pool;

static void register_ort_allocators_once(int device_id) {
  static bool inited = false;
  if (inited) return;
  // 1) 准备池（根据你的配置决定容量/块大小）
  g_host_pool = std::make_unique<core::HostBufferPool>(/*capacity*/16, /*bytes_per_block*/4<<20);
  g_gpu_pool  = std::make_unique<core::GpuBufferPool >(/*capacity*/16, /*w*/0, /*h*/0, core::PixelFormat::RGBA32F);

  // 2) 注册到 Env（CPU Pinned + CUDA 两个 allocator）
  PoolAllocator* pin_alloc = nullptr;
  PoolAllocator* gpu_alloc = nullptr;
  ORT_THROW_ON_ERROR(CreateAndRegisterCpuPinnedAllocator(g_env, &pin_alloc, g_host_pool.get()));
  ORT_THROW_ON_ERROR(CreateAndRegisterCudaAllocator      (g_env, &gpu_alloc,  g_gpu_pool.get(), device_id));
  inited = true;
}

core::Factories build_factories(core::EngineManager& engine,
                                bool use_cuda_source=true, bool use_nvenc=true) {
  // 先把 Env 级 allocator 注册好（只需一次）
  register_ort_allocators_once(/*device_id*/engine.current().device);

  core::Factories F;
  F.make_source = [use_cuda_source](const core::SourceCfg&){
    if (use_cuda_source) return std::static_pointer_cast<ISwitchableSource>(std::make_shared<SwitchableRtspSourceCUDA>());
    return std::static_pointer_cast<ISwitchableSource>(std::make_shared<SwitchableRtspSourceCPU>());
  };
  F.make_filter = [&engine](const core::FilterCfg& c){
    auto A = std::make_shared<analyzer::Analyzer>();
    std::shared_ptr<IPreprocessor> pre =
#ifdef WITH_CUDA
      std::make_shared<LetterboxPreCUDA>(c.in_w, c.in_h);
#else
      std::make_shared<LetterboxPreCPU>(c.in_w, c.in_h);
#endif
    // 使用共享 Env + Options（开启 use_env_allocators）
    OrtModelSession::Options opt{};
    opt.use_iobinding = true;
    opt.use_env_allocators = true;        // ★ 让会话用 Env 注册的 allocator
    opt.cuda_device = engine.current().device;
    // 如需本地复用输出缓冲，也可以设置池大小（可选）
    opt.tensor_device_pool_bytes = 0;     // 依赖 Env allocator 可设 0
    opt.tensor_host_pool_bytes   = 0;

    auto ses = std::make_shared<OrtModelSession>(g_env, c.onnx, opt);

    std::shared_ptr<IPostprocessor> post;
    if (c.task=="det" && c.family=="yolo") post = std::make_shared<YoloV12DetPost>();
    else                                   post = std::make_shared<TransformerDetPost>();
    auto ren = std::make_shared<BoxRenderer>();

    A->swap_pre(pre); A->swap_ses(ses); A->swap_post(post); A->swap_ren(ren);
    return A;
  };
  F.make_encoder = [use_nvenc](const core::EncoderCfg& e){
    if (use_nvenc) return std::static_pointer_cast<IEncoder>(std::make_shared<FfmpegH264EncoderNVENC>());
    return std::static_pointer_cast<IEncoder>(std::make_shared<FfmpegH264EncoderCPU>());
  };
  F.make_transport = [](const core::TransportCfg& t){
    auto tx = std::make_shared<WhipPublisher>(t.whip_url);
    tx->connect(); return tx;
  };
  return F;
}
```

**为什么这么接？**

- **IOBinding** 支持“绑定具体 Ort::Value（预分配）”和“只绑定 MemoryInfo（让会话按域分配）”。后者特别适合**动态输出形状**，而当我们把 allocator 注册到 **Env** 并在会话启用 `use_env_allocators` 后，**会话自己分配**的内存也会直接走你的池，避免额外拷贝。[ONNX Runtime+1](https://onnxruntime.ai/docs/performance/tune-performance/iobinding.html?utm_source=chatgpt.com)
- 设备张量用 `CreateTensor(memory_info="Cuda", device_ptr, ...)` **零拷贝**封装；Host 侧推荐用 **`CudaPinned` 域**实现异步 D2H。[ONNX Runtime](https://onnxruntime.ai/docs/performance/device-tensor.html?utm_source=chatgpt.com)
