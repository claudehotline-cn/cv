我想修改一下流程：用户通过视频源管理界面管理视频源列表，控制VSM开启，停止视频源；用户通过视频分析的页面，第一步使用视频源选择下拉框选择视频源，下拉框提示用户哪些视频源启动，哪些未启动；第二步，通过计算图选择计算图（多阶段分析器）；第三步，启动流。打开分析页面的时候可以默认选择一个开启的视频源和计算图，以便视频自动播放。

很不错，这个流程整体“职责分离”清晰、用户心智负担低，也更贴合你现有多服务架构（VSM 专注源管理，分析页专注分析会话）。下面给你专业评审 + 可落地细化，帮你把坑提前填平。

# 总体评价

- **优点**：
  1. 把“启动/停止视频源”的危险操作隔离到 VSM 页面，分析页只做选择与启动分析，避免误操作；
  2. 下拉框清晰标注 Running/Stopped，降低无效点击；
  3. **默认选中**“首个运行中的源 + 默认计算图”可实现**自动播**，缩短首帧时间；
  4. 非常适合你后端“VSM / GraphRegistry / Orchestrator / Worker”的解耦设计。
- **需要注意的点**：
  - 状态一致性（下拉提示必须是**实时**或**准实时**），否则会出现“看起来 Running 但实际掉线”的错觉。
  - 计算图与源能力的**兼容性校验**（分辨率/像素格式/fps/颜色空间与前处理算子一致）。
  - 浏览器**自动播放策略**（autoplay 需 muted 或用户手势触发）。

# 关键细化建议

## 1) 实时状态同步（VSM → 分析页）

- **推荐**：CP 提供一个**聚合状态订阅**通道（SSE/WebSocket，或 WHEP side channel），分析页订阅后：
  - 源状态：`Running / Starting / Stopped / Error / Degraded`（建议加 **Starting**，避免竞态）。
  - 会话状态：`Preparing(Attach/Load/Warmup) → Running → Stopping → Stopped → Failed`。
- **回退**：若短时间内做不到推送，至少**轮询**（例如 2–3s），并为每条状态设置 TTL（如 5s 过期置灰）。

## 2) 默认自动播放策略

- 打开分析页时：
  1. 拉取 `sources`（含运行状态）、`graphs`（含默认标记）。
  2. 若存在 Running 源且存在可用计算图 → **自动选择**（上次使用优先，其次第一个 Running + 默认图）。
  3. 尝试自动发起 `CreateAnalysis()` 并设置 `<video autoplay muted playsinline>`：
     - **iOS/Safari** 要求 `muted`；如失败，降级为显示“点击播放”大按钮（保留 session）。
- 在 UI 上保留“**自动播放**”开关（本地存储），默认 **ON**，方便测试/演示。

## 3) 源与图的兼容性“预检（preflight）”

- Graph 需要声明能力签名（示例）：

  ```
  {
    "graph_id": "yolo_nms_osd",
    "requires": {
      "color_format": ["NV12","I420"],
      "max_resolution": [3840,2160],
      "min_resolution": [640,360],
      "fps_range": [1,120],
      "batch": [1,4],
      "preprocess": ["resize_letterbox","normalize(0.5,0.5)"]
    }
  }
  ```

- VSM 暴露源能力：

  ```
  {
    "source_id":"cam_01",
    "status":"Running",
    "caps": {
      "codec":"H265",
      "resolution":[1920,1080],
      "fps":30,
      "pix_fmt":"NV12",
      "color_space":"BT.709"
    }
  }
  ```

- 分析页选择时在 CP 做**预检**：不兼容则在下拉项上**禁用**并提示原因（分辨率过高/颜色空间不匹配等）。

## 4) 交互与可用性

- 源下拉项显示**状态徽标**与**轻量指标**（fps、延迟），Stopped 项置灰且 hover 提示“请在视频源管理页启动”。
- 可以提供一个**“去启动”**的链接按钮，跳转到 VSM 页面（保留分析页的上下文）。
  - 若你坚持严格隔离，也可在配置中关闭此快捷入口。
- “启动流”按钮点击后，显示**进度条**分段：Attach Source → Load Graph → Warmup → First Frame。

## 5) 后端接口（建议草案）

**Source/VSM**

- `GET /v1/sources?include_status=true` → 列出源与能力/状态
- `POST /v1/sources/{id}:start` / `:stop`（仅 VSM-UI 调用）
- **推送**：`/v1/stream/sse/sources`（SSE，广播状态变更）

**Graph/Registry**

- `GET /v1/graphs` → 计算图列表 + 默认标记 + 能力要求
- `GET /v1/graphs/{id}` → 图定义（节点/边/参数）

**Analysis/Orchestrator**

- `POST /v1/analysis`

  ```
  {
    "source_id":"cam_01",
    "graph_id":"yolo_nms_osd",
    "params":{"score_thresh":0.25},
    "idempotency_key":"<uuid>"  // 防重复创建
  }
  ```

- `GET /v1/analysis/{session_id}` → 状态机（Preparing/Running/…）+ WHEP/RTC 描述

- `DELETE /v1/analysis/{session_id}` → 停止

- **推送**：`/v1/stream/sse/analysis/{session_id}`（或 WebSocket）

- **鉴权**：分析会话返回**短期令牌**用于 WHEP/WHIP 握手。

## 6) 会话幂等与并发

- 前端“启动流”要加**幂等键**，后端按 `(user, source, graph, idempotency_key)` 去重，避免连点导致重复会话。
- 若存在同源同图的**已运行会话**：可直接复用或返回提示“已在运行（跳转播放）”。

## 7) Nacos/服务发现

- CP 只暴露一个网关；后端内部通过 **Nacos** 发现 `vsm-service`、`graph-registry`、`orchestrator`、`webrtc-gateway`。
- 建议把**计算图配置**、**默认图**、**自动播放开关**做成 **Nacos 动态配置**（支持灰度）。

## 8) 可观测性

- Prometheus 指标建议：
  - `source_status{source_id}`（0/1）、`source_fps`、`source_decode_errors_total`
  - `session_start_latency_seconds`（分阶段 label：attach/load/warmup）
  - `webrtc_first_frame_seconds`、`webrtc_ice_time_seconds`
  - `gpu_mem_bytes{stage}`、`pipeline_fps{stage}`
- 结合 trace（OpenTelemetry）：打通 `ANL-UI → CP → OR → W → RTC` 链路。

## 9) 自动播放与浏览器策略

- `<video>` 设置 `autoplay muted playsinline`，首次失败则弹**一键播放**大按钮（符合策略）。
- 若需要带声，先静音自动播，用户点“开声”再 `unmute`。
- 对 iOS 的 WHEP/RTC 连接/编解码兼容性做 A/B 兜底（必要时走 HLS 预览作为备用）。

## 10) 测试清单（提炼）

- **状态一致性**：VSM 刚把源拉起/刚掉线时，分析页下拉是否**准实时**更新。
- **不兼容拦截**：选择 4K 源 + 仅支撑 1080p 的图，按钮应禁用并说明原因。
- **并发点击**：“启动流”狂点是否只创建一个会话。
- **失败回退**：Attach 失败、Graph 加载失败、Warmup 超时、ICE 失败各自的 UI 提示与重试按钮。
- **自动播放**：多浏览器（Chrome/Edge/Safari/iOS）策略差异覆盖。
- **默认选择**：当没有 Running 源或没有图时，空态界面是否友好（给出“去 VSM 启动 / 去新建图”的 CTA）。

------

## 简要状态机（文本化）

**SourceStatus**：`Stopped → Starting → Running → Degraded/Error → Stopped`
 **SessionStatus**：`Init → Preparing(Attach,Load,Prewarm) → Running → Stopping → Stopped/Failed`

## 前端实现要点（Element Plus）

- 源下拉：`el-select` + `el-tag`（Running=success，Stopped=info/gray，Error=danger）。
- 图下拉：带 **默认**角标（`el-badge`），并在不兼容情况下 `disabled` + `el-tooltip`。
- “启动流”按钮动态状态：`loading`（展示分阶段进度条）。
- `<video>`：`autoplay muted playsinline`，失败后显示遮罩按钮恢复。

```
// src/api.ts
import { ElMessage } from "element-plus";

const BASE = import.meta.env.VITE_API_BASE ?? "https://api.example.com";
const TOKEN = () => localStorage.getItem("token") ?? "";

export type SourceStatus = "Running" | "Stopped" | "Starting" | "Degraded" | "Error";
export interface SourceCaps {
  codec: "H264" | "H265" | "AV1" | "Other";
  resolution: [number, number];
  fps: number;
  pix_fmt: string;
  color_space: string;
}
export interface Source {
  id: string;
  name: string;
  status: SourceStatus;
  caps: SourceCaps;
}

export interface GraphRequires {
  color_format?: string[];
  max_resolution?: [number, number];
  min_resolution?: [number, number];
  fps_range?: [number, number];
  batch?: number[];
  preprocess?: string[];
}
export interface Graph {
  graph_id: string;
  name: string;
  is_default: boolean;
  requires: GraphRequires;
}

export type AnalysisState = "Init" | "Preparing" | "Running" | "Stopping" | "Stopped" | "Failed";
export type AnalysisPhase = "AttachSource" | "LoadGraph" | "Warmup" | "FirstFrame" | "Steady";
export interface AnalysisSession {
  session_id: string;
  state: AnalysisState;
  phase?: AnalysisPhase;
  source_id: string;
  graph_id: string;
  webrtc?: {
    whep_url: string;
    auth_token?: string;
    ice_servers?: Array<{ urls: string[]; username?: string; credential?: string }>;
  };
}

async function http<T>(path: string, init?: RequestInit): Promise<T> {
  const res = await fetch(`${BASE}${path}`, {
    ...init,
    headers: {
      "Content-Type": "application/json",
      Authorization: `Bearer ${TOKEN()}`,
      ...(init?.headers ?? {}),
    },
  });
  if (!res.ok) {
    const err = await res.json().catch(() => ({}));
    ElMessage.error(err?.message || `HTTP ${res.status}`);
    throw new Error(err?.message || `HTTP ${res.status}`);
  }
  return (await res.json()) as T;
}

export const api = {
  listSources: () => http<{ items: Source[] }>(`/v1/sources?include_status=true&status=All`),
  startSource: (id: string) => http<Source>(`/v1/sources/${id}:start`, { method: "POST" }),
  stopSource: (id: string) => http<Source>(`/v1/sources/${id}:stop`, { method: "POST" }),
  listGraphs: () => http<{ items: Graph[] }>(`/v1/graphs`),
  createAnalysis: (body: { source_id: string; graph_id: string; params?: Record<string, any> }, idem?: string) =>
    http<AnalysisSession>(`/v1/analysis`, {
      method: "POST",
      headers: { "Idempotency-Key": idem ?? crypto.randomUUID() },
      body: JSON.stringify(body),
    }),
  getAnalysis: (sid: string) => http<AnalysisSession>(`/v1/analysis/${sid}`),
  stopAnalysis: (sid: string) => http<void>(`/v1/analysis/${sid}`, { method: "DELETE" }),

  // SSE helpers
  sseSources: () => new EventSource(`${BASE}/v1/stream/sse/sources`, { withCredentials: false }),
  sseAnalysis: (sid: string) => new EventSource(`${BASE}/v1/stream/sse/analysis/${sid}`, { withCredentials: false }),
};
```

```
<script setup lang="ts">
import { onMounted, ref, computed, watch } from "vue";
import { ElMessage } from "element-plus";
import { api, type Source, type Graph, type AnalysisPhase, type AnalysisState } from "@/api";

const sources = ref<Source[]>([]);
const graphs = ref<Graph[]>([]);
const selectedSourceId = ref<string>();
const selectedGraphId = ref<string>();
const autoplay = ref<boolean>(localStorage.getItem("autoplay") !== "0");
const videoEl = ref<HTMLVideoElement | null>(null);

const sessionId = ref<string>("");
const sessionState = ref<AnalysisState>("Init");
const sessionPhase = ref<AnalysisPhase | undefined>(undefined);
const starting = ref(false);
const autoplayFailed = ref(false);
let analysisSse: EventSource | null = null;

// derived
const currentSource = computed(() => sources.value.find(s => s.id === selectedSourceId.value));
const canStart = computed(() => !!selectedSourceId.value && !!selectedGraphId.value && currentSource.value?.status === "Running");

function statusTagType(s: Source["status"]) {
  return s === "Running" ? "success" : s === "Starting" ? "warning" : s === "Error" ? "danger" : "info";
}
function gLabel(g: Graph) {
  return g.is_default ? `${g.name}（默认）` : g.name;
}
function isGraphCompatible(g: Graph, s?: Source) {
  if (!s) return true;
  const req = g.requires || {};
  if (req.max_resolution) {
    if (s.caps.resolution[0] > req.max_resolution[0] || s.caps.resolution[1] > req.max_resolution[1]) return false;
  }
  if (req.min_resolution) {
    if (s.caps.resolution[0] < req.min_resolution[0] || s.caps.resolution[1] < req.min_resolution[1]) return false;
  }
  if (req.fps_range) {
    if (s.caps.fps < req.fps_range[0] || s.caps.fps > req.fps_range[1]) return false;
  }
  if (req.color_format && !req.color_format.includes(s.caps.pix_fmt)) return false;
  return true;
}
function saveAutoplay() {
  localStorage.setItem("autoplay", autoplay.value ? "1" : "0");
}

function defaultPick() {
  // 首选 Running 源 & 默认图
  const running = sources.value.find(s => s.status === "Running");
  if (running) selectedSourceId.value = running.id;
  const def = graphs.value.find(g => g.is_default) ?? graphs.value[0];
  if (def) selectedGraphId.value = def.graph_id;
}

async function loadInitial() {
  const [srcRes, graphRes] = await Promise.all([api.listSources(), api.listGraphs()]);
  sources.value = srcRes.items;
  graphs.value = graphRes.items;
  defaultPick();
  if (autoplay.value && canStart.value) {
    await startSession();
  }
}

async function startSession() {
  if (!canStart.value) {
    ElMessage.warning("请选择运行中的视频源与可用计算图");
    return;
  }
  starting.value = true;
  try {
    const idem = crypto.randomUUID();
    const sess = await api.createAnalysis(
      { source_id: selectedSourceId.value!, graph_id: selectedGraphId.value! },
      idem
    );
    sessionId.value = sess.session_id;
    sessionState.value = sess.state;
    sessionPhase.value = sess.phase;
    // 订阅 SSE
    analysisSse?.close();
    analysisSse = api.sseAnalysis(sessionId.value);
    analysisSse.onmessage = (e) => {
      try {
        const s = JSON.parse(e.data) as { state: AnalysisState; phase?: AnalysisPhase };
        sessionState.value = s.state;
        sessionPhase.value = s.phase;
      } catch {}
    };
    // WebRTC/WHEP 播放
    await startWhepPlayback(sess);
  } catch (e: any) {
    // 若返回 409，提示去 VSM 启动源
    // 提示已由 http() 处理 toast
  } finally {
    starting.value = false;
  }
}

async function stopSession() {
  if (!sessionId.value) return;
  try { await api.stopAnalysis(sessionId.value); } catch {}
  analysisSse?.close(); analysisSse = null;
  sessionId.value = ""; sessionState.value = "Stopped"; sessionPhase.value = undefined;
}

function onSourceChange() {
  // 选择 Stopped 源时按钮会禁用；如果有会话在播放，可提示是否切换
}

const stepOrder: AnalysisPhase[] = ["AttachSource","LoadGraph","Warmup","FirstFrame","Steady"];
const stepIndex = computed(() => Math.max(0, stepOrder.indexOf(sessionPhase.value || "AttachSource")));

async function startWhepPlayback(sess: Awaited<ReturnType<typeof api.createAnalysis>>) {
  const whep = sess.webrtc;
  if (!whep || !whep.whep_url) {
    ElMessage.error("WHEP 信息缺失");
    return;
  }
  const pc = new RTCPeerConnection({ iceServers: whep.ice_servers || [] });
  pc.addEventListener("track", (ev) => {
    if (videoEl.value) videoEl.value.srcObject = ev.streams[0];
  });
  // WHEP: recvonly
  const transceiver = pc.addTransceiver("video", { direction: "recvonly" });
  void transceiver; // 仅视频，若需要音频可再加

  const offer = await pc.createOffer();
  await pc.setLocalDescription(offer);

  const resp = await fetch(whep.whep_url, {
    method: "POST",
    headers: {
      "Content-Type": "application/sdp",
      ...(whep.auth_token ? { Authorization: `Bearer ${whep.auth_token}` } : {}),
    },
    body: offer.sdp || "",
  });
  if (!resp.ok) {
    ElMessage.error(`WHEP 握手失败：${resp.status}`);
    return;
  }
  const answerSdp = await resp.text();
  await pc.setRemoteDescription({ type: "answer", sdp: answerSdp });

  // 处理自动播放策略
  const v = videoEl.value!;
  try {
    await v.play();
    autoplayFailed.value = false;
  } catch {
    autoplayFailed.value = true;
  }
}

function resumePlay() {
  videoEl.value?.play().then(() => (autoplayFailed.value = false));
}

onMounted(() => {
  loadInitial();
});
</script>
```

## 备注与落地提示

- **自动播放策略**：代码里已默认 `<video autoplay muted playsinline>`，若浏览器拒绝自动播放，会出现“点击播放”遮罩。
- **兼容性预检**：`isGraphCompatible()` 只做了基础校验，可在后端 `CreateAnalysis` 前再二次校验，返回清晰错误。
- **幂等键**：`Idempotency-Key` 由前端生成 `UUID`；后端以 `(user, source_id, graph_id, idem)` 去重。
- **SSE 事件格式**：建议统一 payload 为 JSON（如 `{ "id": "...", "status": "Running" }` / `{ "state": "...", "phase": "..." }`）。
- **WHEP**：示例使用标准 HTTP SDP 交换；如你使用 WHIP/WHEP 网关（如 Janus/mediasoup/ovenmedia/LiveKit WHEP），请对应调整 URL/鉴权头。
