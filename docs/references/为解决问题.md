崩溃根因

- Windows 下 REST 简易服务器将 WinSock SOCKET 句柄存入 int，在 socket/accept/send/recv/closesocket 的混用路径上存在句柄截断与类型不一
    致，SSE 长连（/api/subscriptions/:id/events 等）高概率触发崩溃。
- 代码位置（示例）
  - video-analyzer/src/server/rest.cpp:276
  - video-analyzer/src/server/rest.cpp:347
  - video-analyzer/src/server/rest.cpp:359
  - video-analyzer/src/server/rest.cpp:573
- 现象：构建阶段或 SSE 写入/关闭时进程异常退出；编译器有 C4244 “SOCKET→int 可能丢失数据”的警告。

  最小修复方案（建议）

- 平台别名统一：在 SimpleHttpServer 内定义
  - Windows：using sock_t = SOCKET; constexpr sock_t INVALID_SOCK = INVALID_SOCKET;
  - POSIX：using sock_t = int; constexpr sock_t INVALID_SOCK = -1;
- 全面替换接口与成员
  - 将 server_socket_、client_socket、handleClient(int)、sendAll(int,...)、SSE 辅助函数参数 fd 改为 sock_t。
  - 判定与重置：全部改用 INVALID_SOCK。
  - 路由处 addStreamRoute 的 lambda 参数 (int fd, const HttpRequest&) 也改为 (sock_t fd, ...)，避免再次强转。
- 旁路增强（可选）
  - 为 SSE 连接开启 TCP keepalive 与合理超时。
  - 在 handleClient 已设置 SO_RCVTIMEO/SO_SNDTIMEO，可保留。
