懂你说的痛点：默认堆出来的 Element Plus UI 很“企业内网风”，离专业的视频监控/视频分析系统还差一截。下面给你一套**可直接替换的设计体系 + 主题 + 组件用法规范 + 关键页面代码骨架**，目标是：**工业级深色主题、专业信息密度、清晰的层次和状态语义**。你把这些落到现有项目里，观感会立刻“提质提色”。

# 0. 总体风格与目标

- **基调**：工业深色（暗蓝/石墨），高对比但不刺眼；卡片层次明显，视频内容“主角化”。
- **信息密度**：表格紧凑、小尺寸控件；仪表指标清晰读数；状态“绿跑、黄告、红错”一眼明了。
- **交互**：轻动效（悬停/聚焦），操作有反馈（加载、禁用、回滚）。
- **元素**：方圆 8px；阴影收敛；图标统一（Element Plus 图标）。

```
import { WhepClient } from './WhepClient'
import { ref, watch, onMounted, onBeforeUnmount } from 'vue'
const props = defineProps<{ whepUrl: string; autoplay?: boolean }>()
const v = ref<HTMLVideoElement|null>(null)
const c = new WhepClient()
async function start(){ if(!props.whepUrl) return; await c.start({ whepUrl: props.whepUrl }, (ms)=>{ if(v.value){ v.value.srcObject=ms; v.value.play().catch(()=>{}) } }) }
async function stop(){ await c.stop(); if(v.value) v.value.srcObject=null }
async function refresh(){ await stop(); await start() }
watch(()=>props.whepUrl, async(nv,ov)=>{ if(nv && nv!==ov) await refresh() })
onMounted(()=> props.autoplay!==false && start())
onBeforeUnmount(()=> stop())
```

# 6. 交互规范（Do / Don’t）

**Do**

- 统一深色背景 + 冷蓝主色；状态色仅用于**状态**，不要满屏彩虹。
- 表格**小号**、紧凑；列对齐，数值右对齐、标签居中。
- 视频为主角：卡片留白、控条半透明，角标显示关键 KPI。

**Don’t**

- 不要大面积纯黑/纯白块；不要在深色背景上使用饱和纯红纯绿大面积填充。
- 不要在同一区域堆 3 种以上字体大小；不使用默认巨大圆角/阴影。
- 不要把“危险”按钮做成主色；危险始终用 `type="danger"` 链接式或次按钮。

```
<script setup lang="ts">
import { ref, watch, onMounted, onBeforeUnmount } from 'vue'
import * as echarts from 'echarts/core'
import { LineChart, BarChart } from 'echarts/charts'
import {
  GridComponent, TooltipComponent, LegendComponent,
  DataZoomComponent, TitleComponent
} from 'echarts/components'
import { CanvasRenderer } from 'echarts/renderers'
import type { ECharts, EChartsOption } from 'echarts'

echarts.use([LineChart, BarChart, GridComponent, TooltipComponent, LegendComponent, DataZoomComponent, TitleComponent, CanvasRenderer])

const props = defineProps<{
  option: EChartsOption
  autoresize?: boolean
  loading?: boolean
}>()

const chartRef = ref<HTMLDivElement | null>(null)
let chart: ECharts | null = null
let ro: ResizeObserver | null = null

function init() {
  if (!chartRef.value) return
  chart = echarts.init(chartRef.value, undefined, { renderer: 'canvas' })
  props.option && chart.setOption(props.option as EChartsOption, { notMerge: true })
  if (props.loading) chart.showLoading('default')
  if (props.autoresize !== false) {
    ro = new ResizeObserver(() => chart?.resize())
    ro.observe(chartRef.value)
  }
}
function dispose() {
  ro?.disconnect(); ro = null
  if (chart) { chart.dispose(); chart = null }
}

watch(() => props.option, (opt) => {
  if (!chart) return
  chart.setOption(opt as EChartsOption, { notMerge: true })
})

watch(() => props.loading, (v) => {
  if (!chart) return
  v ? chart.showLoading('default') : chart.hideLoading()
})

onMounted(init)
onBeforeUnmount(dispose)
</script>

<template>
  <div class="chart-base">
    <div ref="chartRef" class="canvas"></div>
  </div>
</template>

.chart-base { width: 100%; height: 100%; }
```

```
export const cp = {
  // ...你已有的方法
  async metricsQuery(params: { metric:string; from:number; to:number; stepSec:number; pipeline?:string }) {
    const q = new URLSearchParams({
      m: params.metric,
      from: String(params.from),
      to: String(params.to),
      step: String(params.stepSec),
      ...(params.pipeline ? { pipeline: params.pipeline } : {})
    })
    const r = await fetch(`${import.meta.env.VITE_CP_BASE_URL}/metrics/query?`+q.toString())
    if (!r.ok) throw new Error('metricsQuery failed'); return r.json()
  },
  async metricsTop(params: { metric:string; limit:number }) {
    const q = new URLSearchParams({ m: params.metric, limit: String(params.limit) })
    const r = await fetch(`${import.meta.env.VITE_CP_BASE_URL}/metrics/top?`+q.toString())
    if (!r.ok) throw new Error('metricsTop failed'); return r.json()
  },
  async attachSource(payload: { attach_id:string; source_uri:string; pipeline_id:string; options?:Record<string,string> }) {
    const r = await fetch(`${import.meta.env.VITE_CP_BASE_URL}/sources:attach`, {
      method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify(payload)
    })
    if (!r.ok) throw new Error('attachSource failed'); return r.json()
  }
}
```

## 3) DAG → Pipeline Spec 转换与校验（`src/utils/graph.ts`）

> 提供三类能力：
>  ① **校验**（单源/单汇、是否有环、是否连通）
>  ② **拓扑排序**（Kahn）
>  ③ **两种导出**：线性 `toLinearSpec`（兼容你现有的数组式 spec）与 DAG `toDagSpec`（支持边）

```
// src/utils/graph.ts
export interface GEGraph {
  nodes: Array<{ id: string; type: string; name?: string; params?: Record<string, any> }>;
  edges: Array<{ source: string; target: string }>;
}
export interface ValidationResult {
  ok: boolean;
  errors: string[];
  warnings: string[];
}

export function validateGraph(g: GEGraph): ValidationResult {
  const errors: string[] = []
  const warnings: string[] = []

  if (!g.nodes?.length) errors.push('图为空（无节点）')
  const idset = new Set(g.nodes.map(n => n.id))
  for (const e of g.edges || []) {
    if (!idset.has(e.source)) errors.push(`边的 source 缺失: ${e.source}`)
    if (!idset.has(e.target)) errors.push(`边的 target 缺失: ${e.target}`)
  }

  const sources = g.nodes.filter(n => n.type === 'source')
  const sinks = g.nodes.filter(n => n.type === 'sink')
  if (sources.length !== 1) errors.push(`需要且仅需要 1 个 source，当前 ${sources.length}`)
  if (sinks.length !== 1) errors.push(`需要且仅需要 1 个 sink，当前 ${sinks.length}`)

  // 有环检测（Kahn）
  const { order, hasCycle } = topoOrder(g)
  if (hasCycle) errors.push('DAG 检测到环，请检查连线')

  // 连通性（至少 source→sink 可达）
  if (sources.length === 1 && sinks.length === 1) {
    const ok = reachable(g, sources[0].id, sinks[0].id)
    if (!ok) errors.push('sink 不可从 source 到达（图不连通或方向错误）')
  }

  if (g.nodes.length > order.length) warnings.push('存在与主连通分量无关的孤立节点')
  return { ok: errors.length === 0, errors, warnings }
}

export function topoOrder(g: GEGraph): { order: string[]; hasCycle: boolean } {
  const indeg = new Map<string, number>()
  const adj = new Map<string, string[]>()
  g.nodes.forEach(n => { indeg.set(n.id, 0); adj.set(n.id, []) })
  for (const e of g.edges || []) {
    adj.get(e.source)!.push(e.target)
    indeg.set(e.target, (indeg.get(e.target) || 0) + 1)
  }
  const q: string[] = []
  indeg.forEach((d, id) => d === 0 && q.push(id))
  const order: string[] = []
  while (q.length) {
    const u = q.shift()!
    order.push(u)
    for (const v of adj.get(u) || []) {
      const d = (indeg.get(v) || 0) - 1
      indeg.set(v, d)
      if (d === 0) q.push(v)
    }
  }
  const hasCycle = order.length !== g.nodes.length
  return { order, hasCycle }
}

export function reachable(g: GEGraph, src: string, dst: string): boolean {
  const adj = new Map<string, string[]>()
  g.nodes.forEach(n => adj.set(n.id, []))
  for (const e of g.edges || []) adj.get(e.source)!.push(e.target)
  const vis = new Set<string>(); const stk = [src]
  while (stk.length) {
    const u = stk.pop()!; if (u === dst) return true
    if (vis.has(u)) continue; vis.add(u)
    for (const v of adj.get(u) || []) stk.push(v)
  }
  return false
}

/** 主路径（source→sink），用作线性 spec 导出。优先更长的路径。*/
export function findMainPath(g: GEGraph): string[] {
  const src = g.nodes.find(n => n.type === 'source')?.id
  const sink = g.nodes.find(n => n.type === 'sink')?.id
  if (!src || !sink) return []
  // DAG 最长路径（简化：拓扑序上的 DP）
  const { order } = topoOrder(g)
  const pos = new Map(order.map((id, i) => [id, i]))
  // 建邻接
  const adj = new Map<string, string[]>()
  g.nodes.forEach(n => adj.set(n.id, []))
  for (const e of g.edges) adj.get(e.source)!.push(e.target)
  const dist = new Map<string, number>(); const prev = new Map<string, string | null>()
  for (const id of order) { dist.set(id, -Infinity); prev.set(id, null) }
  dist.set(src, 0)
  for (const id of order) {
    for (const v of adj.get(id) || []) {
      if (dist.get(id)! + 1 > dist.get(v)!) { dist.set(v, dist.get(id)! + 1); prev.set(v, id) }
    }
  }
  if (dist.get(sink)! === -Infinity) return []
  const path: string[] = []
  for (let cur: string | null = sink; cur; cur = prev.get(cur) || null) path.push(cur)
  return path.reverse()
}

/** 线性导出：兼容你先前的 nodes[] 数组式 spec。*/
export function toLinearSpec(g: GEGraph, name: string) {
  const vr = validateGraph(g)
  if (!vr.ok) throw new Error('图校验失败：' + vr.errors.join('; '))
  const nodesById = new Map(g.nodes.map(n => [n.id, n]))
  const path = findMainPath(g)
  if (!path.length) throw new Error('未能找到 source→sink 的主路径')
  const srcNode = nodesById.get(path[0])!
  const linearNodes = path.slice(1).map(id => {
    const n = nodesById.get(id)!
    // 类型映射（按你后端约定自行细化；这里保持原 type）
    return { name: n.name || n.type, type: n.type, params: n.params || {} }
  })
  // source 节点如果提供 RTSP/HTTP，走 source_ref；否则留空或由后端解析
  const sourceRef = srcNode.params?.uri || ''
  return { name, source_ref: sourceRef, nodes: linearNodes }
}

/** DAG 导出：如果你的后端支持带 edges 的 Pipeline Spec，推荐用这个。*/
export function toDagSpec(g: GEGraph, name: string) {
  const vr = validateGraph(g)
  if (!vr.ok) throw new Error('图校验失败：' + vr.errors.join('; '))
  const nodes = g.nodes.map(n => ({
    id: n.id,
    name: n.name || n.type,
    type: n.type,
    params: n.params || {}
  }))
  const edges = g.edges.map(e => ({ from: e.source, to: e.target }))
  const srcNode = g.nodes.find(n => n.type === 'source')
  const sourceRef = srcNode?.params?.uri || ''
  return { name, source_ref: sourceRef, nodes, edges }
}
```

### 使用提示与校验建议

- 如果你的后端当前**只接受线性 nodes[]**，先用 `toLinearSpec()`；当后端支持 DAG（带 edges）时，切换 `toDagSpec()` 无缝过渡。
- `validateGraph()` 的规则我只做了“单源/单汇/环/可达/孤立”最小集，你可以按后端约束再加：
  - **节点必填参数**（如 `modelUri`）
  - **类型兼容**（source→preprocess→model→… 的边类型合法性）
  - **多 sink 场景**或**分支/汇合**（若要支持多输出，放宽“单 sink”规则并在 UI 显示）
