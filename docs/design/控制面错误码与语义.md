控制面错误码与语义（VA + VSM）

适用范围
- VideoAnalyzer（VA）控制面：REST 与 gRPC
- VideoSourceManager（VSM）控制面：REST 与 gRPC

统一错误码
- OK：操作成功（对应 REST 200/201/204；gRPC Status OK）
- INVALID_ARG：参数缺失/非法（REST 400；gRPC INVALID_ARGUMENT）
- NOT_FOUND：对象不存在（REST 404；gRPC NOT_FOUND）
- ALREADY_EXISTS：对象已存在/重复创建（REST 409；gRPC ALREADY_EXISTS）
- UNAVAILABLE：资源当前不可用（队列已满/并发限制/外部依赖临时失败）（REST 503；gRPC UNAVAILABLE）
- INTERNAL：未分类内部错误（REST 500；gRPC INTERNAL）

REST 返回结构
- 成功
  - 形如：{"success":true,"code":"OK","data":{...}}
- 失败
  - 形如：{"success":false,"code":"INVALID_ARG","message":"..."}
- HTTP 与 code 的对应关系
  - 200/201/204 -> OK
  - 400 -> INVALID_ARG
  - 404 -> NOT_FOUND
  - 409 -> ALREADY_EXISTS
  - 503 -> UNAVAILABLE
  - 500 -> INTERNAL

gRPC 返回结构
- 成功：Status OK，Reply 内布尔/数据字段有效
- 失败：直接返回标准 StatusCode（INVALID_ARGUMENT/NOT_FOUND/ALREADY_EXISTS/UNAVAILABLE/INTERNAL），客户端以 Status 为准，不再读取 Reply 布尔字段

典型场景与语义
- 创建/附加（Create/Attach）
  - 成功：OK，返回 id/关键字段
  - 已存在：ALREADY_EXISTS（幂等重试可接受，但建议先查询）
  - 参数缺失：INVALID_ARG
  - 依赖不可用（例如拉流器初始化失败）：UNAVAILABLE（退避重试）
- 更新（Update）
  - 成功：OK（幂等操作）
  - 对象不存在：NOT_FOUND
  - 参数缺失/非法：INVALID_ARG
- 删除/分离（Delete/Detach）
  - 成功：OK（幂等）
  - 对象不存在：NOT_FOUND（可视上下文吞错）
- 订阅/切源/切模（VA 控制面）
  - 成功：OK
  - profile/model/uri 缺失：INVALID_ARG
  - 运行态不满足或资源未就绪：UNAVAILABLE（退避重试）
  - 未知异常：INTERNAL
- SSE 并发上限（VSM）
  - 超出：UNAVAILABLE（REST 429，payload code=UNAVAILABLE；建议退避重连）

客户端建议（重试与幂等）
- INVALID_ARG：立即修正参数，勿重试
- NOT_FOUND：读操作可视为软失败；写操作建议先 List/Describe 再补偿
- ALREADY_EXISTS：创建类操作可视为成功（根据业务幂等约定），或用 Update 重试
- UNAVAILABLE：指数退避 + 抖动，直至达到重试上限
- INTERNAL：短暂退避（1~2 次）后上报人工处理

日志与指标（排障建议）
- 服务日志：所有失败均带 code/message，模块前缀 control/app/composition 便于筛选
- VA /metrics：va_cp_auto_* 计数器（控制面自动编排）
- VSM /metrics：vsm_stream_*（源健康），vsm_sse_*（SSE 并发与拒绝）

示例（REST）
- 订阅失败（缺少 profile）
  - 状态：400
  - body：{"success":false,"code":"INVALID_ARG","message":"Missing required field: profile"}
- SSE 并发超限
  - 状态：429
  - body：{"success":false,"code":"UNAVAILABLE","message":"too many sse connections"}

示例（gRPC）
- ApplyPipeline 缺少 graph_id/yaml_path
  - Status：INVALID_ARGUMENT（message: missing graph_id/yaml_path）
- SetEngine 切换失败（设备占用）
  - Status：UNAVAILABLE（message: ...）

版本与兼容性
- 本错误码约定自 IOBinding 分支起统一；旧客户端若只依赖 HTTP 状态/StatusCode，可无缝对接
- 建议新客户端同时解析 code/message，便于前端 UI 与告警平台消费
