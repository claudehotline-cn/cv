# Memo（2026-01-27）：审计模块开始应用

## 目标

- 将已抽离的 `agent-audit` 在实际运行链路中启用：API/Worker/插件节点统一产出审计事件，默认落到 Redis Streams `audit.events`，后续可继续扩展 Kafka 等 transport。

## 关键改动

- `AuditCallbackHandler` 补齐 ChatModel 回调（`on_chat_model_start/end/error`），避免运行时出现 `NotImplementedError(on_chat_model_start)` 并覆盖 ChatModel 事件。
- `@node_wrapper` 支持运行时解析 emitter，避免插件在 import 时创建 Redis 连接导致 event loop 冲突：
  - 优先读取 `config["audit_emitter"]`
  - 其次读取 `config["metadata"]["audit_emitter"]`
  - 未提供时只注入 span 上下文，不发 node 事件（不影响业务执行）。
- API/Worker 统一注入：
  - `agent-platform-api` 的 chat 路由在 `RunnableConfig` 增加 `audit_emitter`，并注入 `callbacks=[AuditCallbackHandler(...)]`。
  - ARQ Worker 执行任务时同样创建 `AuditEmitter + AuditCallbackHandler` 并写入 `config["audit_emitter"]`，使插件节点（LangGraph node）与 LLM/tool 回调事件一致走同一条审计链路。

## 复用方式（其他项目）

- 运行时侧：
  - `emitter = AuditEmitter(redis=redis_async_client)`
  - `config["audit_emitter"] = emitter`
  - `config["callbacks"] += [AuditCallbackHandler(emitter=emitter)]`
  - `config["metadata"]` 内至少包含 `request_id/session_id/thread_id`（用于 DB 归档与关联）。
- 插件侧：
  - `@node_wrapper(..., graph_id=...)` 不再要求模块级 emitter；从上游 `config["audit_emitter"]` 获取即可。

  - `/audit/runs/{request_id}/summary` 中 `spans.type == "node"` 可见（如 `list_tables/table_schema/llm_generate_sql/run_sql/format_output`）。

  - 若 `parent_span_id` 的 parent 尚未落库，先写入 `meta["pending_parent_id"]`，待 parent 出现后 `_adopt_orphans()` 统一补链路。
  - `agent-worker` 重启后不再出现 `ForeignKeyViolationError`。
